🌐 Browser API Polyfills

QTests provides browser API polyfills using happy-dom for lightweight, modern browser environment simulation in Node.js tests.

🪟 Window & Document

Full browser window and document simulation for DOM testing.

import { initializePolyfills, getWindow } from 'qtests'; // Initialize polyfills await initializePolyfills(); // Access browser APIs const window = getWindow(); const doc = window.document; doc.body.innerHTML = '<div id="app"></div>';

📋 Clipboard API

Mock clipboard operations for testing copy/paste functionality.

import { clipboard } from 'qtests'; // Write to clipboard await clipboard.writeText('Test content'); // Read from clipboard const text = await clipboard.readText(); console.log(text); // 'Test content'

📱 Media Queries

Test responsive designs with matchMedia support.

import { matchMedia } from 'qtests'; // Test media queries const query = matchMedia('(min-width: 768px)'); console.log(query.matches); // true/false

🔄 Reset Polyfills

Reset polyfill state for test isolation.

import { resetPolyfills } from 'qtests'; afterEach(() => { // Clean up between tests resetPolyfills(); });

Available Polyfill APIs

  • ✅ initializePolyfills() - Initialize happy-dom environment
  • ✅ getWindow() - Get current window instance

  • ✅ matchMedia(query) - Media query testing

📁 File System Utilities

Safe file system operations with built-in error handling, size limits, and path validation using fs-extra.

📖 Safe File Reading

Read files with automatic error handling and size limits (100MB max).

import { safeReadFile, safeReadFileBuffer } from 'qtests'; // Read as UTF-8 text const content = await safeReadFile('./config.json'); // Read as buffer (for binary files) const buffer = await safeReadFileBuffer('./image.png');

✍️ Safe File Writing

Write files with automatic directory creation and error handling.

import { safeWriteFile, ensureDir } from 'qtests'; // Ensure directory exists await ensureDir('./output/logs'); // Write file safely await safeWriteFile('./output/result.json', JSON.stringify(data, null, 2));

🔍 Existence Check

Check whether a file or directory exists without throwing on missing paths.

import { safeExists } from 'qtests'; const exists = await safeExists('./config.json'); if (!exists) { console.log('Config not found, using defaults'); }

🗑️ Safe Deletion

Delete files or directories with error handling — no throw if already absent.

import { safeDelete } from 'qtests'; // Removes file or directory tree await safeDelete('./test-output/'); await safeDelete('./temp.json');

File System API

  • ✅ safeReadFile(path) — UTF-8 text, 100MB limit
  • ✅ safeReadFileBuffer(path) — binary buffer read
  • ✅ safeWriteFile(path, content) — auto-creates parent dirs
  • ✅ ensureDir(path) — mkdir -p equivalent
  • ✅ safeExists(path) — non-throwing existence check
  • ✅ safeDelete(path) — non-throwing file/dir removal

⚡ Circuit Breaker

Protect your application from cascading failures with the opossum-based circuit breaker implementation.

🔌 Create Circuit Breaker

Configure failure thresholds, timeouts, and recovery behavior.

import { createCircuitBreaker, CircuitState } from 'qtests'; const breaker = createCircuitBreaker({ failureThreshold: 5, resetTimeout: 60000, timeout: 10000, onStateChange: (state) => { console.log('Circuit state:', state); } });

🛡️ Execute with Protection

Wrap async operations with circuit breaker protection.

import { executeWithCircuitBreaker } from 'qtests'; const result = await executeWithCircuitBreaker( async () => { return await fetchExternalAPI(); }, { timeout: 5000, failureThreshold: 3 } );

📊 Monitor Stats

Track success rates, failure counts, and response times.

import { getCircuitBreakerStats } from 'qtests'; const stats = getCircuitBreakerStats(breaker); console.log({ state: stats.state, successRate: stats.successRate, failureCount: stats.failureCount, avgResponseTime: stats.averageResponseTime });

🔗 Connection Pool Manager

Efficient connection management using generic-pool with health monitoring, circuit breaker integration, and automatic recovery.

🏊 Create Connection Pool

Configure pool size, timeouts, and health monitoring.

import { AdvancedConnectionPool } from 'qtests'; const pool = new AdvancedConnectionPool({ create: async () => new DatabaseConnection(), destroy: async (conn) => conn.close(), validate: async (conn) => conn.isAlive() }, { maxConnections: 10, acquireTimeout: 30000, healthCheckInterval: 30000 });

🔄 Acquire & Release

Get connections from the pool and release them when done.

// Acquire a connection const connection = await pool.acquire(); try { // Use the connection await connection.query('SELECT * FROM users'); } finally { // Always release back to pool await pool.release(connection); }

📈 Pool Statistics

Monitor pool health and performance metrics.

const stats = pool.getStats(); console.log({ activeConnections: stats.activeConnections, waitingRequests: stats.waitingRequests, successRate: stats.successfulAcquisitions / stats.totalAcquisitions * 100 });

🛑 Graceful Shutdown

Properly close all connections and clean up resources.

// Graceful shutdown await pool.shutdown(); // Pool events pool.on('connection-acquired', (info) => { }); pool.on('connection-released', (info) => { }); pool.on('error', (error) => { });

💾 Cache Management

High-performance caching with local node-cache and distributed Redis support for optimal performance.

🗄️ Local Cache

In-memory caching with TTL support.

import { createLocalCache } from 'qtests'; const cache = createLocalCache({ defaultTTL: 600, // 10 minutes maxSize: 1000, // Max entries checkperiod: 120 // Cleanup interval }); cache.set('user:123', userData); const user = cache.get('user:123');

🌐 Distributed Cache

Redis-backed caching with local fallback.

import { CacheManager } from 'qtests'; const cache = new CacheManager({ enableDistributed: true, keyPrefix: 'myapp:', defaultTTL: 3600 }); await cache.set('session:abc', sessionData); const session = await cache.get('session:abc');

Delete individual keys or clear entire cache.

// Delete specific key await cache.delete('user:123'); // Clear all cache await cache.clear(); // Graceful shutdown await cache.shutdown();

🧹 Cleanup Manager

Centralized resource management for background tasks, intervals, and cleanup operations to prevent memory leaks.

📋 Register Cleanup Tasks

Register cleanup callbacks for proper resource disposal.

import { globalCleanupManager } from 'qtests'; globalCleanupManager.registerTask( 'db-connection-cleanup', 'Database Connection Cleanup', async () => { await database.close(); }, { frequency: 60000, component: 'database' } );

▶️ Task Management

Enable, disable, and trigger cleanup tasks.

// Enable/disable task globalCleanupManager.enableTask('task-id'); globalCleanupManager.disableTask('task-id'); // Trigger immediate cleanup await globalCleanupManager.runTask('task-id'); // Unregister task globalCleanupManager.unregisterTask('task-id');

📊 Cleanup Metrics

Monitor cleanup operations and resource usage.

const metrics = globalCleanupManager.getMetrics(); console.log({ totalTasks: metrics.totalTasks, activeTasks: metrics.activeTasks, completedCleanups: metrics.completedCleanups, failedCleanups: metrics.failedCleanups });

🛑 Graceful Shutdown

Shutdown all tasks and run final cleanup.

// In your app shutdown handler process.on('SIGTERM', async () => { await globalCleanupManager.shutdown(); process.exit(0); }); // Or create custom manager import { createCleanupManager } from 'qtests'; const manager = createCleanupManager({ defaultCleanupInterval: 60000, enableMetrics: true });
